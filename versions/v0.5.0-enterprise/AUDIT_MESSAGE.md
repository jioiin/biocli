Я даю тебе полный исходный код проекта BioPipe-CLI. Контекст проекта — в файле PROJECT_CONTEXT.md, прочитай его первым.

Твоя задача — провести полный аудит кода по двум направлениям: качество кода и безопасность. Работай как principal engineer, который ревьюирует код перед релизом в production.

## Что делать

### Шаг 1: Прочитай PROJECT_CONTEXT.md
Пойми архитектуру, security model, и инварианты проекта. Запомни главный принцип: LLM — недоверенный генератор текста, вся безопасность — в Python-коде, не в промптах.

### Шаг 2: Проверь безопасность (по SECURITY_AUDIT.md)
Пройди по каждому пункту чеклиста из SECURITY_AUDIT.md. Для каждого пункта:
- Найди код (файл, строка)
- Verdict: PASS / FAIL / PARTIAL
- Если FAIL — напиши fix (конкретный код)

Особое внимание:
- Может ли что-то обойти 10-слойный SafetyValidator?
- Может ли плагин изменить Config, PermissionPolicy, или allowlist?
- Может ли session.restore() загрузить инжектированные SYSTEM messages?
- Есть ли код который делает сетевые вызовы во время работы (не при setup)?
- Зависит ли какое-то решение о безопасности от ответа LLM?

### Шаг 3: Проверь качество кода (8 категорий)

**Категория 1 — Заглушки:** найди функции с pass, ..., hardcoded return, try/except pass, TODO/FIXME, мёртвый код.

**Категория 2 — Сложность:** функции > 30 строк, вложенность > 3, дублирование, магические числа, переменные < 3 символов.

**Категория 3 — Типизация:** функции без type hints, Any где можно конкретный тип, type: ignore без причины, Optional без обработки None.

**Категория 4 — Ошибки:** bare except, глотание исключений, отсутствие валидации входов, ресурсы без with, async без CancelledError.

**Категория 5 — Архитектура:** circular imports, God classes, tight coupling, мутабельное глобальное состояние, core/ импортирует из llm/rag/ напрямую.

**Категория 6 — Тесты:** модули без тестов, тесты без assertions, тесты зависящие от порядка, отсутствие edge cases.

**Категория 7 — Документация:** публичные функции без docstring, docstring на русском (должен быть английский), несоответствие docstring и кода.

**Категория 8 — Производительность:** list вместо set для поиска, конкатенация строк в цикле, regex compile внутри цикла, sync I/O в async.
